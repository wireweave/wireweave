# 다국어 동기화

영문 (en) 이 원문, 한국어 (ko) / 일본어 (ja) 가 번역.

## 디렉토리 구조

```
src/
  guide/intro.md             # en (default locale)
  ko/guide/intro.md          # 한국어
  ja/guide/intro.md          # 일본어
```

## 동기화 정책

영문 변경 시:

1. 같은 PR 에 한국어 / 일본어 동시 갱신
2. 시간이 부족하면 placeholder 마커 + 후속 PR 약속:

```md
---
title: New Feature
needs_translation: true
---

# 새 기능

> 이 페이지는 영문에서 업데이트되었습니다. 번역 진행 중.

(영문 원문 그대로 유지 또는 기계 번역 placeholder)
```

`needs_translation: true` frontmatter 가 있는 페이지는 사이트에서 배지 표시 + sitemap 에서 후순위.

## 용어 일관성

용어집 (glossary):

| EN             | KO            | JA               |
| -------------- | ------------- | ---------------- |
| Wireframe      | 와이어프레임  | ワイヤーフレーム |
| Component      | 컴포넌트      | コンポーネント   |
| Bounded layout | 경계 레이아웃 | 境界レイアウト   |
| Renderer       | 렌더러        | レンダラー       |

`src/.vitepress/glossary.json` 에 정의. 신규 용어 추가 시 PR 에서 함께 업데이트.

## 코드 예제

코드 블록 안의 변수명 / 주석은 번역하지 않는다 (실행 검증 깨짐):

```wireframe
# OK — 주석 번역
# 한국어 주석

# Bad — 변수/속성 번역
Button:확인  ← 영문 원문 'Submit' 유지
```

## URL slug

영문 slug 가 기준. 한/일 페이지도 같은 slug 사용:

```
/guide/getting-started      ← en
/ko/guide/getting-started   ← ko
/ja/guide/getting-started   ← ja
```

`/ko/시작하기` 같은 한국어 slug 금지 — URL 공유 / SEO 일관성.

## 검색

VitePress local search 는 locale 별 인덱스 분리. en / ko / ja 검색 결과가 섞이지 않음.

## 절대 금지

- en 만 갱신하고 ko / ja 영구 미갱신 (drift)
- ko 페이지에 한국어 slug
- 코드 예제의 변수 / 속성 / 컴포넌트 이름 번역
- en 과 ko / ja 의 헤딩 hierarchy 가 다름 (자동 anchor link 깨짐)
- 자동 기계 번역만으로 게시 (사람 검수 필요 — 최소 1회)
